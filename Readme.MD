## Observações aferidas de uma desenvolvedora assando no forno:
- Não se deve usar entidades de caracteres HTML ao definir os parâmetros para JS, nessas funções onclick insert;

- HTML é uma linguagem de marcação, JS é uma linguagem de programação, então, logo, sinais matemáticos só são apenas símbolos para o primeiro e operadores aritméticos para o outro;

- Eval, função, avalia expressão e retorna resultado, mais ou menos bom para funções internas do código, sem input externo. Ruim para comunicação com servidor(malware e fácil usabilidade de outros users).

---

### Mudanças no code
1. Uso de entidades de caracteres para HTML;
2. ERRO MASCARADO -> Overflow: hidden; para esconder números excedentes;
3. Title definido no parágrafo para fazer graça e reflexão;
4. Personalização própria de estilo no CSS, com boder-radius, box-shadown, etc.
